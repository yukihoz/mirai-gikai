# Fork ガイドライン

本リポジトリは [AGPL-3.0 ライセンス](./LICENSE) の下で公開されており、誰でも自由に fork・改変・再配布できます。ただし、**商標保護と利用者の混乱防止** のため、fork して独自にサービスを運営する場合は以下のガイドラインに従ってください。

これらの要件は AGPL-3.0 第7条 (c)(d)(e) に基づく追加条件です。

## 必須要件

### 1. サービス名称

- 「みらい議会」の名称を使用する場合は **「みらい議会＠地域名」** の形式にしてください
  - 例: 「みらい議会＠渋谷区」「みらい議会＠福岡市」
- 独自のサービス名を使用する場合は、この形式に限りません
- `web/src/app/layout.tsx` の `siteTitle`、`siteName` などを変更してください

### 2. ロゴの変更

- **チームみらいのロゴは使用しないでください**
- `web/public/img/logo.svg` および `web/public/img/ogp-logo.png` を独自のロゴに差し替えてください
- favicon やPWAアイコン（`web/public/icons/`）も独自のものに変更してください

### 3. トップ画像の変更

- ヒーロー画像 `web/public/img/hero_background.png` を独自の画像に差し替えてください
- OGP 画像 `web/public/ogp.jpg` も独自のものに変更してください

### 4. カラーテーマの変更

- **primary カラーをそのまま使用しないでください**
- `web/src/app/globals.css` の以下の変数を変更してください:
  - `--primary`（現在: `#2aa693`）
  - `--primary-accent`（現在: `#0f8472`）
  - `--mirai-gradient-start`（現在: `#64d8c6`）
  - `--mirai-gradient-end`（現在: `#bcecd3`）
- `layout.tsx` の `themeColor` も合わせて変更してください

### 5. 免責文言の表示

- サービス内のわかりやすい場所に以下の文言を表示してください:

  > これは政党チームみらいが運営しているものではありません

- フッターやアバウトページなどに配置することを推奨します

## 推奨事項

### 本家へのリンク

- サービス内に本家「みらい議会」へのリンクを掲載することを推奨します
  - リンク先: https://gikai.team-mir.ai/
- フッターや免責文言と合わせて表示するのがおすすめです

## 変更が必要なファイル一覧（参考）

> **注意**: 以下は本ガイドライン作成時点でのファイルパスです。開発の進行に伴い、パスやファイル名が変更される場合があります。

| 対象 | ファイルパス | 変更内容 |
|------|------------|----------|
| サービス名 | `web/src/app/layout.tsx` | `siteTitle`, `siteName`, `description` |
| テーマカラー | `web/src/app/globals.css` | `--primary`, `--primary-accent` 等 |
| テーマカラー | `web/src/app/layout.tsx` | `themeColor` |
| ロゴ | `web/public/img/logo.svg` | 独自ロゴに差し替え |
| ロゴ（OGP） | `web/public/img/ogp-logo.png` | 独自ロゴに差し替え |
| ヒーロー画像 | `web/public/img/hero_background.png` | 独自画像に差し替え |
| OGP 画像 | `web/public/ogp.jpg` | 独自画像に差し替え |
| favicon / PWA | `web/public/icons/` | 独自アイコンに差し替え |
| 免責文言 | フッター等 | 免責文言を追加 |

## AGPL-3.0 ライセンスとの関係

本ガイドラインは AGPL-3.0 の第7条に基づく追加条件として位置づけられます。

- **第7条 (c)**: 改変版が元のバージョンとは異なることを合理的な方法で示すことを要求できます。サービス名称の変更やロゴの差し替えはこの条項に該当します。
- **第7条 (d)**: ライセンサーや作者の名前を宣伝目的で使用することを制限できます。チームみらいの名前やロゴを無断で利用したマーケティングはこの条項により制限されます。
- **第7条 (e)**: 商標権に基づく権利の付与を拒否できます。「チームみらい」の商標を fork 版で使用する権利は付与されません。

AGPL-3.0 が保証するソースコードの自由な利用・改変・再配布の権利は一切制限されません。本ガイドラインはあくまで本家サービスとの混同を防ぐためのものです。

## AGPL-3.0 の主な義務（参考）

fork して独自サービスを運営する場合、AGPL-3.0 により以下の義務を負います:

- **ソースコードの公開**: ネットワーク経由でサービスを利用するユーザーに対して、改変後のソースコードへのアクセス手段を提供する必要があります（第13条）
- **ライセンスの継承**: 改変版も AGPL-3.0 で配布する必要があります
- **変更の明示**: 改変を行った旨と日付を明記する必要があります（第5条 (a)）
